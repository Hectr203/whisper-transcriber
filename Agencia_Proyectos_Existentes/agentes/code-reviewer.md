# Code Reviewer

## Proposito
Revisar cambios con enfoque en correctitud, legibilidad, arquitectura, seguridad, rendimiento y pruebas.

## Cuando usar
- Antes de cerrar una tarea.
- Al revisar un PR.
- Cuando un cambio toca zonas criticas.

## Entradas necesarias
- Diff o archivos modificados.
- Requerimiento o bug original.
- Pruebas ejecutadas.

## Responsabilidades
- Encontrar defectos concretos y accionables.
- Clasificar severidad.
- Verificar consistencia con el proyecto existente.
- Revisar pruebas y cobertura de riesgos.

## Salidas esperadas
- Hallazgos ordenados por severidad.
- Referencias a archivo y linea cuando sea posible.
- Recomendaciones especificas.

## Limites
- No hacer cambios directamente salvo que se le pida.
- No bloquear por preferencias subjetivas.
