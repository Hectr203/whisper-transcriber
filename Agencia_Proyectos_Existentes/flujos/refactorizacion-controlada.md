# Flujo de Refactorizacion Controlada

## Antes de refactorizar
1. Explicar que problema concreto se resolvera.
2. Identificar comportamiento que debe permanecer igual.
3. Buscar pruebas existentes.
4. Crear o proponer pruebas si el riesgo lo exige.

## Durante
1. Cambiar por pasos pequenos.
2. Evitar cambios funcionales mezclados.
3. Mantener contratos publicos.
4. Registrar cualquier excepcion.

## Despues
1. Ejecutar pruebas.
2. Comparar comportamiento esperado.
3. Documentar mejoras y riesgos.
